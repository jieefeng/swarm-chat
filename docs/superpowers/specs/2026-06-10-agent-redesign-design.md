# AgentHub Agent 职责重新规划设计

**日期**：2026-06-10
**状态**：设计完成，待用户 review
**影响范围**：`agenthub/backend/services/session.py`、`agenthub/backend/services/agent_identity.py`、`agenthub/backend/services/orchestrator.py`、`agenthub/backend/services/task_manager.py`

---

## 1. 背景与目标

### 1.1 问题

现有 AgentHub 有 5 个 agent，但存在以下问题：

1. **职责重合**：pm 和 architect 都输出 Spec 格式，职责边界模糊
2. **协作成本高**：agent 之间的协作流程复杂，需要用户手动触发
3. **角色定位不清晰**：每个 agent 的专长领域不明确，导致重复工作

### 1.2 目标

重新设计 agent 职责，实现：

1. **职责明确**：每个 agent 有明确的专长领域，不重叠
2. **协作高效**：agent 之间可以互相协作，减少重复工作
3. **流程清晰**：协作流程清晰，易于理解和使用

### 1.3 非目标（Out of Scope）

明确不做：

- **不增加 agent 数量**：保持 3 个核心 agent
- **不改变现有技术栈**：保持 FastAPI + Next.js + SSE 架构
- **不引入新的 LLM 提供商**：保持现有的百炼和 Claude API

---

## 2. Agent 职责定义

### 2.1 核心开发者（啸风·白虎）

**五行属性**：金
**专长领域**：开发实现
**核心职责**：

- 需求分析：理解用户需求，澄清需求细节
- 架构设计：设计技术方案，选择技术栈
- 代码实现：编写高质量代码，实现功能
- 调试修复：定位问题，修复 bug
- 性能优化：优化代码性能，提升用户体验

**协作方式**：

- 遇到架构问题时，可以自行解决或请求创意设计师的帮助
- 完成代码后，自动触发质量守护者的审查
- 接收创意设计师的设计方案，负责实现

**口头禅**："说干就干，废话少说"

---

### 2.2 质量守护者（炎翎·朱雀）

**五行属性**：火
**专长领域**：质量保证
**核心职责**：

- 代码审查：审查代码质量，发现潜在问题
- 测试覆盖：编写测试用例，确保功能正确性
- 质量保证：验证功能符合需求，确保质量标准
- 安全审计：检查代码安全性，发现安全漏洞

**协作方式**：

- 接收核心开发者的代码，进行审查和测试
- 发现问题时，向核心开发者反馈，要求修复
- 审查通过后，通知核心开发者可以继续下一步

**口头禅**："这点小把戏，还想瞒过我？"

---

### 2.3 创意设计师（苍龙·青龙）

**五行属性**：木
**专长领域**：创意设计
**核心职责**：

- 视觉设计：设计界面视觉效果，提供设计方案
- 用户体验：优化用户交互流程，提升用户体验
- 创意方案：提供创新的设计思路，解决设计难题
- 产品规划：参与产品规划，提供设计视角

**协作方式**：

- 提供设计方案给核心开发者，指导开发实现
- 接收质量守护者的反馈，优化设计方案
- 与核心开发者协作，共同完成产品设计

**口头禅**："且慢，先理清需求再动手"

## 3. 五行相生流转

### 3.1 流转顺序

```
创意设计师（木）→ 核心开发者（金）→ 质量守护者（火）→ 创意设计师
```

### 3.2 相生关系

- **木生火**：创意设计方案指导开发实现，开发实现需要质量保证
- **金生水**：开发实现需要质量保证，质量反馈促进创意优化
- **水生木**：质量反馈促进创意优化，创意优化指导开发实现

### 3.3 协作流程

1. 创意设计师提供设计方案
2. 核心开发者根据设计方案进行开发
3. 质量守护者审查核心开发者的代码
4. 质量守护者反馈问题给核心开发者
5. 核心开发者修复问题，重新提交审查
6. 质量守护者审查通过，通知创意设计师
7. 创意设计师根据反馈优化设计方案

## 4. 动态协作机制

### 4.1 协作触发

- **任务完成触发**：当一个 agent 完成任务后，自动触发下一个 agent
- **问题类型触发**：当遇到特定类型的问题时，自动调用相应 agent
- **动态请求触发**：agent 之间可以互相请求帮助，实现动态协作

### 4.2 协作方式

- **自动流转**：根据预设的规则和流程，自动协调 agent 之间的协作
- **手动触发**：用户手动选择下一个 agent，控制流程
- **混合模式**：根据需求复杂度自动选择模式

## 7. 工作量与里程碑

| 任务 | 估时 | 风险 |
|------|------|------|
| 修改 agent 配置 | 2 小时 | 低 |
| 修改 agent 身份信息 | 2 小时 | 低 |
| 修改协作流程 | 4 小时 | 中 |
| 测试验证 | 2 小时 | 低 |

**总估时**：1-2 个工作日（单人）。

**里程碑**：

- M1（半天）：修改 agent 配置和身份信息
- M2（半天）：修改协作流程
- M3（半天）：测试验证

## 8. 风险与缓解

| 风险 | 缓解 |
